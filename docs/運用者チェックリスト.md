## A. OCR を実データで動かす本番セットアップ（最優先）

> なぜ私（Claude）ではできないか: あなたの GCP 課金アカウント・認証情報が必要な作業のため。
> コードはすべて用意済みです。**職員ログイン（旧 A-8）もコード実装済み**になり、
> 旧 A-1〜A-8・A-10 は **`scripts/setup-gcp.sh` が対話式で一括実行**します。
> 残るあなたの作業は実質 4 手です。

- [ ] **1. 課金つきの GCP を準備**（唯一スクリプト化できない部分）
  - プロジェクト自体はスクリプトが作成できます。**課金アカウントの紐付けだけ**コンソールで行ってください
    （https://console.cloud.google.com/billing）
- [ ] **2. あなたの PC で 1 コマンド実行**
  ```bash
  gcloud auth login && firebase login   # 初回のみ
  bash scripts/setup-gcp.sh
  ```
  これで以下がすべて自動で終わります（何度実行しても安全）:
  - API 有効化 / Firebase 追加 / Firestore・Storage 作成（旧 2）
  - Document AI Form Parser プロセッサ作成（旧 3）
  - IAM 付与 ※Functions v2 の実行 SA（`PROJECT_NUMBER-compute@…`）にも付与するよう修正済み（旧 4）
  - `.firebaserc`・`functions/.env` の生成（旧 5）、CORS の本番 URL 限定（旧 10）
  - `npm install && npm test` → `firebase deploy`（旧 6）、関数 URL の表示（旧 7）
  - メール／パスワード ログインの有効化と**職員アカウントの作成**（旧 8 のコンソール作業）
  - 最後に **GitHub Secrets に貼る値がそのまま表示されます**
- [ ] **3. GitHub Secrets を登録して再デプロイ**（旧 9）
  - リポジトリ → Settings → Secrets and variables → Actions → New repository secret
    - `OCR_ENDPOINT` … スクリプトが表示した関数 URL
    - `FIREBASE_CONFIG` … スクリプトが表示した 1 行 JSON
    - `OCR_API_KEY` … 任意（使う場合のみ）
  - Actions → 「Deploy to GitHub Pages」→ Run workflow（`deploy.yml` への配線は済んでいます）
- [ ] **4. 動作確認**（旧 11）
  - SPA を開く → **職員ログイン画面**が出る → 作成したアカウントでログイン
  - ヘッダーに緑の「**本番接続**」バッジ、取り込み画面に「ライブ取り込み」が出れば成功

> 手動でやりたい場合・スクリプトが途中で失敗した場合の個別手順は
> [`OCR-BACKEND.md`](./OCR-BACKEND.md) に全文あります（失敗したステップだけ手動で済ませて再実行すれば OK）。
