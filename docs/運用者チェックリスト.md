## A. OCR を実データで動かす本番セットアップ（最優先）

> なぜ私（Claude）ではできないか: あなたの GCP 課金アカウント・認証情報が必要な作業のため。
> コードはすべて用意済みで、「認証情報を入れてデプロイ」するだけの状態です。

- [ ] **1. GCP プロジェクトを用意**（課金を有効化）
- [ ] **2. API を有効化**（Document AI / Cloud Functions / Cloud Build / Run / Firestore / Storage）
- [ ] **3. Document AI プロセッサを作成**（まず Form Parser、ロケーション `us` 等）→ プロセッサ ID を控える
- [ ] **4. IAM 付与**: Functions 実行 SA（`PROJECT@appspot.gserviceaccount.com`）に `roles/documentai.apiUser`
- [ ] **5. 設定ファイル**: `.firebaserc`（プロジェクト ID）と `functions/.env`（`DOCAI_*`）を記入
- [ ] **6. デプロイ**: `cd functions && npm install && npm test` → `firebase deploy --only functions,firestore:rules,firestore:indexes,storage`
- [ ] **7. 関数 URL を控える**（例 `https://asia-northeast1-PROJECT.cloudfunctions.net/recognizeSheet`）
- [ ] **8. 職員ログイン（Firebase Auth）を導入** ← セキュリティルールが `request.auth` を必須にしているため**本番前に必須**
- [ ] **9. フロントに環境変数を設定**（GitHub リポジトリ Secret → `deploy.yml` の build に `env:` で渡す → 再デプロイ）
  - `VITE_OCR_ENDPOINT`（単票読み取り／フェーズ1）
  - `VITE_FIREBASE_CONFIG`（Firestore キュー／フェーズ2。Web アプリ構成 JSON を 1 行で）
- [ ] **10. `OCR_ALLOW_ORIGIN` を本番 SPA の URL に限定**（`*` のままにしない）
- [ ] **11. 動作確認**: ローカルエミュレータ → 本番。取り込み画面に「実データで読み取り」「ライブ取り込み」が出れば OK
